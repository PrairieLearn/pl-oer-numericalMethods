You are working for Sony developing an emulator for a collection of games released on the original Playstation One. Unfortunately, the 3D graphics engines of old video game consoles used fixed-point arithmetic. Your job is to update the code to use IEEE single precison floating-point numbers.

Suppose you encounter the following binary number:

$$x = {{params.number}}$$

How would you store this number as an IEEE single precision number? Specify the sign, exponent, and significand. Think carefully about how this information is stored. Is the exponent stored directly, or in some other manner? Do all digits of the binary number need to be stored?

Your answers should be strings of 1's and 0's. The significand of a single precision number consists of 23 bits; for this number, the last 17 bits are zero, so you only need to enter the first six bits.

For more info on IEEE single precision numbers, including their representation, please consult the reference page. {{#feedback.question_feedback}}
